Elementos utilizados para el desarrollo de este proyecto
  1. El Algoritmo encargado de la solución cuenta con recursivad para intentar encontrar un camino que complete la duración de la sesión
  2. En la solución se crean tantos Tópicos como sean necesarios para ubicar todas las conferencias
  3. Las Sesiones en esta versión son datos estáticos pero se gestionan dinámicamente, de forma tal que en la próxima versión se pueda implementar una vista donde el usuario los defina, ejemplo crear una nueva sesión (Break) o variar el tiempo de las mismas
  4. Se realizaron validaciones con expresiones regulares para asegurar que el título de la conferencia no contenga números
  5. La entrada del programa es un fichero,se utilizó la función unique para renombrar archivo y asegurar que no se sobreescriba
