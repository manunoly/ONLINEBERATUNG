Solución creada por Manuel Almaguer Código Fuente

  1. La Solución fue desarrollada con PHP orientado a objetos y Angular para el front

  2. Lógica de Paquetes
  3. Modelo de Clases PHP
  4. El Algoritmo encargado de la solución fue realizado con recursividad para intentar encontrar un camino que complete la duración de la sesión
  5. En la solución se crean tantos Tópicos como sean necesarios para ubicar todas las conferencias
  6. Las Sesiones en esta versión son datos estáticos pero se gestionan dinámicamente, de forma tal que en la próxima versión se pueda implementar una vista donde el usuario los defina, ejemplo crear una nueva sesión (Break) o variar el tiempo de las mismas
  7. Se realizaron validaciones con expresiones regulares para asegurar que el título de la conferencia no contenga números
  8. La entrada del programa es un fichero,se utilizó la función unique para renombrar archivo y asegurar que no se sobreescriba
  9. Al asignar una conferencia en una sesión se calcula la hora de inicio de la conferencia según la hora de inicio de la sesión y la duración de las conferencias anteriores a esta en la sesión
  10. Se verifica que exista la carpeta upload y si no existe se crea
  11. Se realiza tratamiento de errores en el front para mostrar siempre una respuesta al usuario
  12. Se crearon controladores para gestionar las peticiones del front y que utilizan el modelo de clases para manejar los datos
  13. El front esta divididos en componentes, contiene un servicio llamado api que se utiliza para el intercambio con el back
  14. El código se transpila desde typescript a javascript y comprime para hacerlo lo más liviano posible
  15. Se incluyó bootsrap para dar estilo a la interfaz
